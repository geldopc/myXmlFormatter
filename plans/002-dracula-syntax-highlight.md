## Why this matters

After plan 001, the view mode shows plain monospace text. Syntax highlighting
(coloring tags, attributes, values, comments) is the single highest-impact UX
improvement for a code formatter — it makes the output immediately scannable.
The desired color scheme is the **Dracula theme**, which is dark-mode native
and pairs well with the existing minimal black-and-white UI.

## Steps

### Step 1: Create `src/utils/xmlHighlight.ts`

Create a new file that exports one function: `highlightXml(xml: string): string`.
It takes a prettified XML string and returns an HTML string with `<span>` elements
for colorizing. The output is safe to use in `dangerouslySetInnerHTML`.

**Critical safety rule**: All text content (the actual data between XML tags)
and attribute values MUST be HTML-escaped before wrapping in spans. Use this
helper at the top of the file:

```typescript
function escapeHtml(s: string): string {
  return s
    .replace(/&/g, "&amp;")
    .replace(/</g, "&lt;")
    .replace(/>/g, "&gt;")
    .replace(/"/g, "&quot;")
    .replace(/'/g, "&#39;");
}
```

**Tokenizer approach** — process the string left-to-right, consuming one token
at a time. Each token type maps to a CSS variable. Use the following priority
order when checking what starts at position `i`:

1. `<!--` → comment (`--xml-color-comment`): scan to `-->` (inclusive)
2. `<![CDATA[` → CDATA (`--xml-color-cdata`): scan to `]]>` (inclusive)
3. `<!DOCTYPE` or `<!doctype` → DOCTYPE (`--xml-color-doctype`): scan to first `>` (inclusive)
4. `<?` → processing instruction (`--xml-color-pi`): scan to `?>` (inclusive)
5. `</` → closing tag: render `</` + tag name + `>` with `--xml-color-punct` for punctuation, `--xml-color-tag` for tag name
6. `<` followed by a letter → opening tag: parse tag name and attributes (see sub-steps below)
7. Anything else → text content: consume until the next `<`, HTML-escape it, wrap with `--xml-color-text`

**Opening tag parsing** (case 6 in detail):
- Consume `<` (punct)
- Consume tag name until whitespace, `/`, or `>` (tag color)
- While not at `>` or `/>`:
  - Skip whitespace
  - Consume attribute name until `=` or whitespace (attr-name color)
  - If `=` found: consume `=` (punct), then consume the quoted value including quotes (attr-value color — HTML-escape the value inside)
- Consume `/>` or `>` (punct)

**Span helper**:
```typescript
function span(cssVar: string, content: string): string {
  return `<span style="color:var(${cssVar})">${content}</span>`;
}
```

The function signature:
```typescript
export function highlightXml(xml: string): string {
  // implementation
}
```

The function must handle malformed input gracefully: if a scan for a closing
token (e.g. `-->`) reaches end-of-string without finding it, treat the rest
of the string as one token of the appropriate type and stop.

**Verify**: The file typechecks with `npx tsc --noEmit` (exit 0). You can also
quick-test in a browser console:
```
import { highlightXml } from "./src/utils/xmlHighlight";
console.log(highlightXml('<root attr="val">text</root>'));
// Must contain <span style="color:var(--xml-color-tag)">root</span>
// and no raw unescaped < or > in text content
```

### Step 2: Add Dracula CSS variables to `src/index.css`

Append the following block at the very end of `src/index.css`,
after all existing content. Do not modify any line above this block.

```css
/* ── Dracula XML syntax highlight ─────────────────── */
:root,
.dark {
  --xml-color-tag: #8be9fd;
  --xml-color-attr-name: #50fa7b;
  --xml-color-attr-value: #f1fa8c;
  --xml-color-text: #f8f8f2;
  --xml-color-comment: #6272a4;
  --xml-color-pi: #ff79c6;
  --xml-color-cdata: #ffb86c;
  --xml-color-doctype: #bd93f9;
  --xml-color-punct: #ff79c6;
}
```

Using `:root, .dark` ensures the variables are available in both light and
dark mode. The Dracula colors are intentionally the same in both modes —
the app's light mode will show these colors on the light background, which
is intentional (the high-contrast Dracula colors remain readable).

**Verify**: Open the browser after this step; no visual regressions in the
app UI (the variables aren't used anywhere yet until Step 3).

### Step 3: Wire `highlightXml` into the view-mode `<pre>` in Home

In `src/pages/Home/index.tsx`:

1. Add import at the top:
   ```tsx
   import { highlightXml } from "@utils/xmlHighlight";
   ```

2. Replace the `<pre>` text child with `dangerouslySetInnerHTML`:

   **Before** (from plan 001):
   ```tsx
   <pre
     id="xml-output"
     tabIndex={0}
     onKeyDown={handleKeyDown}
     onClick={handleBackToEdit}
     onScroll={(e) => syncGutterScroll(e.currentTarget.scrollTop)}
     className="flex-1 overflow-auto pt-8 pb-28 pl-4 pr-6 cursor-text focus:outline-none text-foreground/85 whitespace-pre"
   >
     {input}
   </pre>
   ```

   **After**:
   ```tsx
   <pre
     id="xml-output"
     tabIndex={0}
     onKeyDown={handleKeyDown}
     onClick={handleBackToEdit}
     onScroll={(e) => syncGutterScroll(e.currentTarget.scrollTop)}
     className="flex-1 overflow-auto pt-8 pb-28 pl-4 pr-6 cursor-text focus:outline-none whitespace-pre"
     dangerouslySetInnerHTML={{ __html: highlightXml(input) }}
   />
   ```

   Note: remove `text-foreground/85` from className — the span colors from
   `highlightXml` take over; a base color on the `<pre>` would conflict.

3. The `<pre>` must now be a self-closing tag (no children) because
   `dangerouslySetInnerHTML` and children are mutually exclusive in React.

**Verify**: `npx tsc --noEmit` exits 0. In the browser, paste XML, click
Prettify, and confirm tags appear in cyan, attribute names in green,
attribute values in yellow, and text content in white.

### Step 4: Performance guard — memoize `highlightXml` call

The `<pre>` re-renders on every keystroke while in view mode (React
re-renders the whole component on state changes). The highlight computation
runs every render. Since `input` only changes when the user types (but view
mode shows a read-only `<pre>`), memoize the output:

In `src/pages/Home/index.tsx`, add below the `lineCount` memo:
```tsx
const highlightedXml = React.useMemo(
  () => (viewMode !== "edit" ? highlightXml(input) : ""),
  [input, viewMode]
);
```

Then use `highlightedXml` in the `dangerouslySetInnerHTML`:
```tsx
dangerouslySetInnerHTML={{ __html: highlightedXml }}
```

**Verify**: `npx tsc --noEmit` exits 0.
